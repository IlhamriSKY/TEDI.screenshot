# TEDI Screenshot

One-click window screenshot extension for [TEDI](https://github.com/IlhamriSKY/TEDI).
Adds a camera icon to the status bar (or press `Mod+Alt+S`); the resulting
PNG is copied to the clipboard and saved to your Downloads folder.
A native sidecar handles the capture so WebGL terminal panes render
correctly instead of coming out black.

> [!NOTE]
> The bundled sidecar binary is unsigned. The first launch on each
> platform may show SmartScreen (Windows), Gatekeeper (macOS), or
> nothing (Linux). See [Trust prompts](#trust-prompts).

---

## Install

1. Open **Settings → Extensions** in TEDI.
2. Switch to the **From GitHub** tab.
3. Paste `IlhamriSKY/TEDI.screenshot` and click **Review → Install**.

## Update

In **Settings → Extensions**, click **Check updates** on this extension's
card. If a new release exists, click **Update** and TEDI reinstalls in
place.

## How it works

```
TEDI webview                       Native sidecar
┌────────────────────────┐        ┌──────────────────────────────┐
│ extension.js           │  spawn │ tedi-screenshot-helper       │
│ • status-bar button    │ ─────▶ │ • picks window by title      │
│ • Mod+Alt+S keybinding │        │ • xcap composites the frame  │
│ • collect base64 PNG   │ ◀───── │ • writes base64 PNG to stdout │
└────────────────────────┘        └──────────────────────────────┘
       │                                       │
       ▼                                       ▼
clipboard + Downloads/tedi-<stamp>.png   xcap / Quartz / PipeWire
```

On click the extension:

1. Picks the helper binary for the current OS / arch from
   `sidecar/<platform>-<arch>/`.
2. Spawns it via `shell_bg_spawn_direct` with the TEDI window title.
3. Reads the base64 PNG from stdout via `shell_bg_logs`.
4. Decodes into a `Blob`, writes to the clipboard, triggers a
   `<a download>` for the Downloads folder.

Native deps (`xcap`, `image`, `libpipewire-0.3-dev` on Linux) live in
this repo's `sidecar-src/` crate, not TEDI core. Uninstalling the
extension removes them with it.

## Permissions

| Permission | Why |
| --- | --- |
| `panels:register` | Auto-render the status-bar toggle button from the manifest. |
| `ui:toast` | Surface capture results / errors. |
| `invoke:shell_bg_spawn_direct` | Start the sidecar (no shell wrapper, so the tracked PID is the helper itself). |
| `invoke:shell_bg_logs` | Read base64 PNG from the helper's stdout. |
| `invoke:shell_bg_kill` | Stop the helper on disable / uninstall. |

No filesystem, keychain, or network permissions.

## Trust prompts

| Platform | First launch | How to clear |
| --- | --- | --- |
| Windows | SmartScreen ("Windows protected your PC"). | Click **More info → Run anyway** once. |
| macOS | Gatekeeper may flag the helper as quarantined. | `xattr -dr com.apple.quarantine ~/Library/Application\ Support/id.ilhamrisky.tedi/extensions/tedi.screenshot/sidecar` |
| Linux | Nothing. TEDI's installer `chmod 0755`s `sidecar/` after extraction. | — |

## Development

```bash
git clone https://github.com/IlhamriSKY/TEDI.screenshot.git
cd TEDI.screenshot

# Build the sidecar for your host.
cd sidecar-src
cargo build --release
mkdir -p ../sidecar/<platform>-<arch>      # e.g. linux-x86_64
cp target/release/tedi-screenshot-helper* ../sidecar/<platform>-<arch>/
cd ..

# UI: build extension.js from src/ (generated by esbuild — not committed).
npm install
npm run build

# Package + install via Settings → Extensions → From file:
zip -r dev.zip manifest.json extension.js logo.png README.md CHANGELOG.md LICENSE sidecar
```

To cut a release, tag `vX.Y.Z` and push. The CI in
[`.github/workflows/release.yml`](.github/workflows/release.yml) builds
the sidecar for every supported platform and uploads the zip to the
GitHub release.
